# Plano de Design — Landing Page Estética, Conceitual e Responsiva

## 1. Objetivo visual

A landing page deve transmitir três sensações principais:

- inteligência;
- sofisticação;
- clareza.

O site não deve parecer uma plataforma genérica de cursos, resumos ou inteligência artificial.

A proposta visual deve ser conceitual, marcante e contemporânea, com forte presença editorial e foco em leitura, conhecimento e transformação pessoal.

A experiência deve lembrar uma mistura entre:

- uma revista digital premium;
- uma biblioteca contemporânea;
- um produto de tecnologia elegante;
- uma plataforma de aprendizagem de alto valor.

A página deve causar impacto logo no primeiro contato, mas sem excesso visual.

---

# 2. Direção criativa

## Conceito principal

### Conhecimento que cabe na rotina

O design deve representar a ideia de transformar livros densos em momentos curtos, organizados e valiosos de aprendizagem.

O conceito visual pode combinar:

- blocos editoriais;
- páginas de livro;
- marcações;
- destaques tipográficos;
- pequenas animações;
- camadas de conteúdo;
- cartões de leitura;
- linhas de tempo;
- indicadores de progresso.

## Palavras-chave da identidade

- editorial;
- refinado;
- inteligente;
- moderno;
- memorável;
- humano;
- calmo;
- premium;
- atemporal;
- funcional.

## O que evitar

- excesso de gradientes;
- visual infantil;
- ilustrações genéricas;
- muitos ícones coloridos;
- cards em excesso;
- animações chamativas;
- estética de curso online barato;
- aparência de dashboard técnico;
- excesso de cores vibrantes;
- fundos muito carregados;
- excesso de informações na primeira dobra.

---

# 3. Estrutura visual da página

A landing page deve seguir uma narrativa visual.

A ordem recomendada é:

1. Header;
2. Hero;
3. Seção de dor;
4. Seção de identificação;
5. Seção de solução;
6. Como funciona;
7. Exemplo de microlição;
8. Benefícios;
9. Diferenciais;
10. Prova social;
11. Planos;
12. Perguntas frequentes;
13. CTA final;
14. Rodapé.

Cada seção deve possuir uma função clara, uma composição própria e uma cor ou atmosfera específica.

# 6. Grid e espaçamento

## Grid desktop

```text
12 colunas
largura máxima: 1280px
margens laterais: 48px a 72px
```

## Grid tablet

```text
8 colunas
margens laterais: 32px
```

## Grid mobile

```text
4 colunas
margens laterais: 20px
```

## Sistema de espaçamento

Utilizar uma base de 8px.

Escala recomendada:

```text
8px
16px
24px
32px
48px
64px
96px
128px
```

As seções devem ter bastante espaço vertical.

Desktop:

```text
96px a 160px
```

Mobile:

```text
64px a 96px
```

# 7. Header

## Estrutura

O header deve conter:

- logo;
- link "Como funciona";
- link "Benefícios";
- link "Planos";
- link "Perguntas";
- botão principal.

## Estilo

Fundo:

```text
transparente sobre o hero
```

Ao rolar:

```text
fundo off-white com leve blur
```

Altura:

```text
72px a 88px
```

## Botão principal

Texto:

```text
Quero começar
```

Estilo:

```text
fundo verde profundo
texto branco
border-radius: 999px
padding horizontal amplo
```

## Mobile

No mobile:

- esconder menu completo;
- usar botão de menu;
- manter CTA visível;
- menu abrir em tela cheia;
- links grandes e bem espaçados.

# 8. Hero

## Objetivo

Criar impacto imediato e comunicar a transformação.

## Fundo

```text
Off-white com leve textura de papel
```

Pode haver:

- linhas editoriais;
- números de página;
- elementos de marca;
- palavras flutuantes;
- recortes de textos;
- uma composição abstrata inspirada em páginas de livros.

## Layout desktop

Divisão em duas colunas:

### Coluna esquerda

- selo pequeno;
- headline;
- subheadline;
- CTA principal;
- CTA secundário;
- microprova;
- lista curta de benefícios.

### Coluna direita

- mockup de celular;
- mockup de e-mail;
- cartões de trechos;
- indicador de leitura;
- páginas em camadas.

## Copy sugerida

### Selo

```text
Conhecimento para quem tem pouco tempo
```

### Título

```text
Você não precisa de mais livros.
Precisa de mais tempo para aprender.
```

### Subtítulo

```text
Transforme 20 minutos da sua rotina em aprendizados práticos extraídos de grandes livros.
```

### CTA principal

```text
Quero aprender todos os dias
```

### CTA secundário

```text
Ver uma microlição
```

## Elemento conceitual

Criar uma animação suave mostrando:

```text
300 páginas
↓
20 minutos
↓
1 grande aprendizado
```

## Mobile

No mobile:

- título em no máximo 4 linhas;
- mockup abaixo do CTA;
- CTA em largura total;
- esconder elementos decorativos secundários;
- manter leitura limpa.

# 13. Exemplo de microlição

## Fundo

```text
Preto suave ou verde profundo
```

## Objetivo

Mostrar o produto de forma tangível.

## Layout desktop

Duas colunas:

### Coluna esquerda

- título;
- explicação;
- tópicos do que o usuário recebe;
- CTA.

### Coluna direita

- mockup grande de celular;
- versão de e-mail;
- cartão com trecho do livro;
- barra de progresso.

## Conteúdo do mockup

```text
Hábitos Atômicos

Tempo de leitura: 18 minutos

O que você vai aprender

1. Como pequenos hábitos geram grandes resultados
2. Como o ambiente influencia o comportamento
3. Como construir uma identidade consistente

Trecho do livro

Aplicação prática

Resumo
```

## Estilo do mockup

- fundo claro;
- sombra suave;
- bordas grandes;
- tipografia editorial;
- barras de progresso;
- pequenos detalhes dourados.

## Mobile

Mostrar um único mockup por vez com rolagem horizontal.

```text
Off-white
```

- depoimentos;
- número de leitores;
- número de lições;
- avaliação;

- cards sem borda pesada;
- destaque em aspas;
- tipografia editorial.

# 21. Componentes do site

## Botões

### Primário

```text
fundo verde profundo
texto branco
border-radius: 999px
```

### Secundário

```text
fundo transparente
borda verde profundo
texto verde profundo
```

### Hover

- deslocamento de 2px;
- sombra suave;
- transição de 200ms;
- leve alteração de contraste.

---

# 22. Cards

## Estilo

- bordas de 20px a 28px;
- fundo limpo;
- sombra discreta;
- borda fina;
- bastante padding.

## Evitar

- sombras pesadas;
- bordas muito escuras;
- excesso de cards iguais;
- muitos elementos dentro do mesmo card.

---

# 23. Formulários

## Campos

- altura mínima de 52px;
- bordas arredondadas;
- rótulo claro;
- placeholder discreto;
- estado de foco visível;
- feedback de erro acessível.

## CTA do formulário

Sempre ocupar largura total no mobile.

---

# 24. Animações

## Animações permitidas

- fade-in;
- slide-up;
- parallax suave;
- cards flutuando;
- palavras destacadas;
- barra de progresso;
- páginas se reorganizando;
- transições entre etapas.

## Duração

```text
200ms a 600ms
```

## Regra

A animação deve reforçar a ideia, nunca distrair.

---

# 25. Responsividade

## Breakpoints sugeridos

```text
Mobile pequeno: até 480px
Mobile: até 768px
Tablet: 769px a 1024px
Desktop: 1025px a 1440px
Desktop amplo: acima de 1440px
```

## Mobile

- navegação simplificada;
- CTAs em largura total;
- títulos menores;
- uma coluna;
- cards empilhados;
- mockups com rolagem horizontal;
- menos elementos decorativos;
- espaçamento vertical de 64px;
- botões com altura mínima de 48px.

## Tablet

- duas colunas quando possível;
- cards em grid de 2;
- mockups reduzidos;
- menus adaptados;
- títulos intermediários.

## Desktop

- grid de 12 colunas;
- composições assimétricas;
- grandes áreas em branco;
- mockups lado a lado;
- elementos decorativos completos;
- animações mais ricas.

## Desktop amplo

O conteúdo deve continuar limitado a uma largura máxima.

Nunca esticar textos por toda a tela.

---

# 26. Acessibilidade

O design deve ser bonito e também acessível.

## Requisitos

- contraste adequado;
- textos legíveis;
- fonte mínima de 16px no mobile;
- foco visível;
- navegação por teclado;
- imagens com texto alternativo;
- animações reduzidas para quem preferir;
- botões com área mínima de toque;
- formulários com labels;
- estados de erro claros.

---

# 27. Performance

## Diretrizes

- imagens em WebP ou AVIF;
- lazy loading;
- fontes otimizadas;
- no máximo duas famílias tipográficas;
- animações leves;
- SVG para ícones;
- evitar vídeos pesados no hero;
- carregar elementos decorativos depois do conteúdo principal.

## Meta

A primeira dobra deve carregar rapidamente, especialmente no mobile.
